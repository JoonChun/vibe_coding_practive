# 📄 PRD: MAPF Post-Mortem Analyzer (MAPF 데드락 사후 분석기)

## 1. 프로젝트 개요 (Project Overview)

* **프로젝트명:** MAPF Post-Mortem Analyzer
* **프로젝트 소개:** 대규모 다중 에이전트 경로 탐색(MAPF) 환경에서 발생한 정체 및 데드락(Deadlock) 로그를 시각화하고, 브라우저 내에서 인과관계 분석과 즉각적인 경로 재탐색(Re-plan)을 지원하는 무설치 기반의 웹 샌드박스 툴.
* **해결하려는 문제:** 현장 엔지니어가 로봇 정체 발생 시, 시각 자료 없이 수만 줄의 텍스트 로그(JSON/CSV)에 의존해 원인 제공자를 찾아야 하는 비효율성. 기존 범용 툴이 제공하지 못하는 '알고리즘적 인과관계(우선순위 꼬임 등)' 진단 부재.
* **핵심 가치:** * 에이전트 간의 'Wait-for-Graph(대기 상태 사이클)'를 자동 추출하여 데드락의 근본 원인을 즉시 타겟팅.
  * 서버나 DB 구축 없이 브라우저 단독으로 동작하여, 보안이 중요한 현장에서도 즉시 사용 가능.

## 2. 사용자 시나리오 (User Scenarios)

1. **데드락 원인 추적:** 물류 센터에서 AGV 4대가 엉켰을 때, 엔지니어가 로그를 툴에 드래그 앤 드롭한다. 툴이 즉각적으로 꼬인 로봇 간의 'Wait-for Cycle'을 빨간 화살표로 렌더링하여 최초 원인 제공자를 10초 만에 시각적으로 찾아낸다.
2. **브라우저 내 샌드박스 실험:** 병목 구간에 임시 페널티(가중치)를 부여하고 브라우저 내장 A* 알고리즘으로 Re-plan을 돌려, 실제 서버 적용 전 로봇들의 우회 경로를 안전하게 검증한다.
3. **운동학적 충돌 검증:** 격자(Grid) 상으로는 겹치지 않았으나 현장에서 부딪힌 로봇들의 로그를 재생한다. 로봇의 회전 반경과 실제 크기(Footprint) 폴리곤을 시각화하여, 회전 시 점유 공간이 겹치는 찰나의 순간을 잡아낸다.

## 3. 핵심 기능 목록 (Key Features)

* **[P1] 필수 구현 기능 (MVP):**
  * **로컬 로그 파서:** 50MB 이하 JSON/CSV 로그 파일 브라우저 내 즉시 파싱.
  * **2D 토폴로지 & 렌더러:** 맵 및 에이전트 이동 시각화.
  * **타임라인 컨트롤러:** 재생, 일시정지, 배속, 프레임 이동 및 데드락 마커 표시.
  * **Wait-for-Graph 자동 추출기:** 정지된 에이전트 간의 타겟 노드 점유 상태를 비교해 방향성 그래프(사이클)를 그리고 시각화.
* **[P2] 차별화 기능:**
  * **AI 기반 동적 스키마 어댑터:** 이기종 로봇의 커스텀 JSON 포맷을 Gemini API를 활용해 표준 포맷으로 변환하는 JavaScript 코드를 동적으로 생성 및 적용.
  * **브라우저 내장 샌드박스:** 특정 노드 페널티 부여 및 Web Worker 기반 A* 우회 경로 탐색.
  * **운동학적 검증기:** 에이전트 폴리곤 간의 물리적 충돌(Intersection) 프레임 검사.

## 4. 기술 스택 (Tech Stack)

* **프론트엔드 프레임워크:** React (Vite) + TypeScript
* **상태 관리:** Zustand (재생 프레임, 전역 상태 동기화)
* **렌더링 엔진:** HTML5 Canvas API (배경 및 에이전트) + React Flow / SVG (그래프 오버레이)
* **고비용 연산:** Web Workers API (A* 알고리즘, DFS 사이클 탐색, 폴리곤 연산 병렬 처리)
* **AI 및 서버리스:** Vercel Serverless Functions + Gemini API (스키마 어댑터 생성 및 리포트 작성용 BFF 프록시)
* **배포:** Vercel 또는 GitHub Pages

## 5. 화면 구성 (UI Layout & Wireframe)

* **Layout Structure:** 단일 페이지 애플리케이션 (SPA)
* **영역 A (Main Canvas - 좌측 70%):** 파일 업로드 드롭존, 2D Grid Map, 에이전트 렌더링, Wait-for-Graph 빨간색 화살표 오버레이.
* **영역 B (Timeline - 하단 10%):** 재생/일시정지 버튼, 배속 컨트롤, 이벤트 마커(데드락 지점)가 포함된 타임라인 슬라이더.
* **영역 C (Inspector & AI Sidebar - 우측 20%):** 선택된 에이전트 상태값, 감지된 데드락 사이클 텍스트, [AI 원인 분석] 결과 리포트 출력 패널, 샌드박스(Re-plan) 컨트롤 패널.

## 6. 상세 기능 명세 (Detailed Specifications)

* **Log Parser:** 파일 업로드 즉시 백그라운드에서 전체 프레임을 1회 스캔하여 데드락 발생 타임스탬프를 사전 캐싱. 필수 키값 누락 시 에러 반환.
* **Deadlock Detection (DFS):** 속도가 0인 에이전트들을 필터링 후, A -> B 방향성 간선을 만들고 DFS로 순환(Cycle) 구조를 찾아 외곽선을 하이라이트.
* **Dynamic Schema Adapter:** 처음 보는 포맷의 JSON 업로드 시 첫 50줄을 추출해 Gemini API로 전송. 반환받은 파싱 로직 코드를 샌드박스에서 실행해 나머지 데이터를 로컬 변환.
* **Sandbox Re-planner:** Web Worker를 호출하여 메인 스레드 블로킹 없이 계산 후, 새로운 우회 경로를 점선으로 렌더링. 경로가 없으면 에러 토스트 발생.

## 7. 디자인 가이드 (Design Guide)

* **테마:** 눈의 피로도를 낮추고 데이터 가독성을 높이는 Dark Mode 전용, Flat Design.
* **컬러 시스템:** * Background: Zinc-900 (`#18181B`)
  * Primary: Blue (`#3B82F6`)
  * Semantic: Deadlock = Neon Red (`#EF4444`), Warning/Penalty = Amber (`#F59E0B`), Success = Neon Green (`#10B981`).
* **타이포그래피:** UI는 Pretendard/Inter, 데이터 및 좌표 표기는 고정폭 폰트(JetBrains Mono/Fira Code) 사용.
* **시각화:** 에이전트는 반투명 폴리곤으로 처리하여 충돌 시 교집합이 붉게 블렌딩되도록 설계.

## 8. 제약 사항 (Constraints)

* **클라이언트 제약:** 브라우저 메모리 한계로 단일 로그 파일 크기는 최대 50MB로 제한.
* **성능 제약:** A* 연산 및 폴리곤 충돌 연산은 메인 UI 스레드 멈춤 방지를 위해 반드시 Web Worker 환경에서 격리 실행.
* **AI 컨텍스트 제약:** Gemini API 호출 시 전체 로그 전송 절대 불가. 반드시 데드락 발생 반경 5x5 데이터 및 관련 로봇 상태값만 추출하여 전송.
* **MVP 범위 제약:** 실시간 FMS 연동 불가(오직 사후 분석), 3D 렌더링 배제(엄격한 2D Top-down 유지).
