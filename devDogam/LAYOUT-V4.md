## 3. 새 SVG 구조 (IlwolObongdo.tsx 재작성)

### 3.1 viewBox · preserveAspectRatio

```
viewBox:               0 0 1000 600
preserveAspectRatio:   xMidYMid slice
width / height:        100% / 100%
position:              absolute, inset:0
z-index:               0
pointer-events:        none
```

viewBox는 V3의 `0 0 1000 600`을 그대로 유지한다. 변경 없음.
단, 요소 y좌표를 재배분하여 마룻바닥을 y=500 이하로 올리고 그 위 공간에 계단·카펫을 넣는다.

### 3.2 레이어 z 순서 (아래→위)

```
z=0   배경 한지 베이지 (div 배경색 #F5EDD8)
z=1   [SVG 내부 렌더 순서]
       레이어1: 단청 천장 + 처마 (y=0~50)         ← REFERENCE 핵심 요소 7
       레이어2: 좌·우 기둥 몸체 (x=0~70, x=930~1000)  ← 핵심 요소 6
       레이어3: 일월오봉도 후벽 (x=70~930, y=53~370)   ← 핵심 요소 4
       레이어4: 옥좌 단상 (x=400~600, y=310~380)       ← 핵심 요소 1·3
       레이어5: 빨간 계단 5단 (y=380~430)              ← 핵심 요소 3
       레이어6: 빨간 카펫 사다리꼴 (y=430~500)          ← 핵심 요소 5
       레이어7: 마룻바닥 + 마룻결 (y=500~600)           ← 핵심 요소 8
       레이어8: 기둥 받침 (y=480~500)                  ← 기둥 하단 마감
z=5   도제 캐릭터 (기둥 뒤에서 나오는 연출)
z=8   도제 캐릭터 앞면 (기둥 앞으로 나온 후)

z=20  왕 캐릭터
z=30  말풍선
```

> SVG 내부에서는 나중에 그린 요소가 앞에 렌더됨 (페인터 알고리즘).
> 기둥은 SVG 최상단(첫 rect)에서 그리고 z-index로 분리하지 않고,
> 기둥 위에 캐릭터가 올라오는 연출은 CSS z-index로 처리.

### 왕 top 좌표 산출 근거

어전 영역 CSS height = 전체 800 - 48(헤더) - 64(두루마리) - 56(KingInput) = 632px.
SVG viewBox y=322 (단상 상판 상단) = 322/600 = 53.7% (SVG 내 비율).
어전 영역은 CSS로 `flex-1`이므로 SVG 비율이 CSS 상대 퍼센트와 일치.
`top: 40%` = SVG y≈240 = 왕 발밑이 좌석(y=320) 위에 올라앉는 위치.
`transform: translate(-50%, -100%)` 적용 시 캐릭터 높이(약 64px SVG 환산 ≈40px CSS)만큼 위로 당겨짐.
실제 발 끝 = top 40% + 캐릭터 높이.

### side prop 방향 사유

좌열 (정도전·정약용) `side="right"`: 말풍선 꼬리가 오른쪽(옥좌 방향)을 향함.
우열 (이순신·장영실) `side="left"`: 말풍선 꼬리가 왼쪽(옥좌 방향)을 향함.
두 열 모두 *임금을 향해 보고*하는 방향감 확보.
