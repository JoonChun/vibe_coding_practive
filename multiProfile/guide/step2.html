PRD 작성하기

코딩을 시작하기 전에, 만들고 싶은 것을 체계적으로 정리합니다. Gemini가 도와줄 거예요.

📋 PRD란?

PRD(Product Requirements Document)는 "내가 뭘 만들 건지"를 정리한 기획서입니다. 설계도 없이 집을 지을 수 없듯이, PRD 없이 프로젝트를 시작하면 중간에 길을 잃기 쉽습니다.

좋은 소식은, AI에게 아이디어만 말하면 PRD를 대신 작성해준다는 것입니다!

✍️ Gemini로 PRD 작성하기

Gemini에게 다음과 같은 프롬프트를 입력해보세요. 여러분의 아이디어에 맞게 수정하면 됩니다.

나는 [프로젝트 아이디어]를 만들고 싶어. 타겟 사용자는 [대상]이고, 핵심 목적은 [목적]이야. 이 프로젝트의 PRD를 작성해줘. 다음 항목을 포함해줘: 1. 프로젝트 개요 2. 사용자 시나리오 3. 핵심 기능 목록 4. 기술 스택 (HTML, CSS, JS 기반) 5. 화면 구성 6. 디자인 가이드 (색상, 폰트, 분위기) 7. 제약 사항
💡 팁: 처음에는 완벽하지 않아도 됩니다. AI에게 "이 부분을 더 자세히 써줘" 또는 "이 기능은 빼줘"라고 요청하며 다듬어 가세요.

🏗️ 좋은 PRD의 구조

AI가 작성한 PRD에 다음 내용이 포함되어 있는지 확인하세요:

🎯 프로젝트 개요
무엇을, 왜, 누구를 위해 만드는지
⚙️ 핵심 기능 목록
구체적으로 어떤 기능이 필요한지
🎨 디자인 가이드
색상, 폰트, 전체적인 분위기
🚧 제약 사항
기술적 한계, 예산, 시간 등

📱 실전 예시: vi.joon Link

이 사이트(vi.joon Link)도 PRD를 먼저 작성한 후 만들었습니다. 실제로 사용한 PRD의 일부를 보여드릴게요:

프로젝트 명: vi.joon Link 핵심 컨셉: '바이브 코딩'을 통해 만든 결과물들을 기록하고 일반인에게 그 가능성을 전달하는 성장형 포트폴리오. 기술 스택: HTML5, CSS3, Vanilla JavaScript, Tailwind CSS 디자인: - Main Color: #8A2BE2 (Electric Purple) - Background: #0F1014 (Dark Charcoal) - Style: Glassmorphism + Neon Glow

이렇게 정리된 PRD가 있으면, 다음 단계에서 AI에게 "이 PRD대로 만들어줘"라고 요청할 수 있습니다.
