[PRD] 뉴스 브루 (News Brew) - 개인 맞춤형 뉴스 자동화 비서

1. 프로젝트 개요
프로젝트 이름: 뉴스 브루 (News Brew)

한 줄 설명: 사용자가 설정한 키워드 뉴스를 AI가 정교하게 추출(Brewing)하여 요약하고, 구글 스프레드시트 기록과 이메일 발송을 자동화하는 개인 맞춤형 뉴스 큐레이션 서비스.

유형: 로컬 실행형 웹 애플리케이션

난이도: 중급 (Intermediate)

1. 사용자 시나리오
대상: 특정 산업 트렌드를 추적하는 직장인, 투자자, 정보 수집가.

상황:

설정 시: 웹 UI를 통해 추적할 키워드, 수집 기간, 뉴스 개수, 이메일 수신 시간을 설정.

활용 시: 매일 아침 출근길이나 업무 시작 전, AI가 요약한 리포트를 이메일로 받아 확인.

목적: 정보 과잉 시대에 뉴스 확인 시간을 단축하고, 중요 정보를 체계적으로 아카이빙하기 위함.

사용 흐름: 키워드 등록 → 자동 수집/저장 → AI 중요도 판별 및 요약 → 이메일 배달 → 과거 기록 조회 및 원문 확인.

1. 핵심 기능 목록
필수 기능 (MVP+)
키워드 & 필터링 관리: 키워드 추가/삭제, 특정 언론사 제외(블랙리스트), AI 기반 광고성 기사 필터링.

뉴스 수집 엔진: 검색 API 연동, 수집 기간(최근 24시간/직접 지정 등) 및 수집 수량 제어.

구글 스프레드시트 아카이빙: 수집된 모든 뉴스 원문 데이터(제목, 링크, 날짜) 자동 기록.

AI 뉴스 브루잉: LLM을 활용한 중요도 판단 및 핵심 요약(개조식 3~5줄) 생성.

이메일 자동 배달: Resend API를 통해 커피 브라운 톤의 가독성 좋은 리포트 발송.

과거 요약본 아카이브: 웹 UI에서 날짜별 지난 요약본을 다시 볼 수 있는 히스토리 기능.

상세 실시간 대시보드: 오늘 수집 수, 총 키워드, 마지막 실행 시간 표시 및 백엔드 작업 로그(WebSocket) 실시간 출력.

자동 스케줄러: 로컬 서버 상주 시 지정된 시간에 전체 프로세스 자동 실행.

선택 기능
수동 즉시 실행 버튼 ("지금 브루잉 시작").

이메일 템플릿 테마 설정 기능.

1. 기술 스택
Frontend: React (Vite) / Next.js, Tailwind CSS, Shadcn/UI, Lucide React.

Backend: Python 3.10+, FastAPI, APScheduler, WebSockets.

Database: SQLite (설정 및 아카이브 저장), Google Sheets API (데이터 백업).

External APIs: Google Custom Search API / SerpApi, OpenAI GPT-4o-mini / Gemini API, Resend.

1. 화면 구성
메인 대시보드: 상단 요약 지표 카드, 중앙 실시간 진행 상태 로그 콘솔, 퀵 액션 버튼.

키워드 관리: 키워드 입력 필드, 리스트 테이블, 상세 수집 옵션(기간, 개수, 정렬 기준).

아카이브 페이지: 날짜별 요약본 리스트 및 상세 내용 팝업 뷰.

환경 설정: API Key 관리, 자동화 스케줄 시간 설정, 이메일 수신자 관리.

1. 상세 기능 명세
수집 로직: 설정된 날짜 범위와 개수 제한을 준수하며, AI가 중요도를 1~10점으로 판별하여 기준 미달 기사는 자동 제외.

AI 프로세싱: 최신성(Freshness)을 고려한 프롬프트 설계, 중복 기사 통합 요약, 이메일 하단 주의 문구 포함.

연동 로직: Google 서비스 계정을 통한 시트 행 추가, Resend 발송 실패 시 3회 재시도 로직 적용.

실시간 통신: 백엔드의 각 단계(수집 시작 -> 검색 중 -> 요약 중 -> 발송 완료)를 WebSocket으로 프론트에 실시간 전송.

1. 디자인 가이드
컨셉: Warm & Minimal (따뜻한 에스프레소 바 감성).

컬러 팔레트:

Brew Coffee (#4B3621), Warm Cream (#F5F5DC), Morning Orange (#FF8C00).

텍스트: 가독성을 위해 본문은 고대비 Deep Brown (#1A1A1A) 사용.

타이포그래피: Pretendard (본문), JetBrains Mono (로그 콘솔).

로고: 커피잔에서 신문 모양의 김이 모락모락 피어오르는 엠블럼.

1. 제약 사항
API Quota: Google 검색 API의 일일 100회 무료 할당량 내 운영 필요.

환경 의존성: PC가 켜져 있어야 스케줄러가 작동하는 로컬 앱의 한계 존재.

품질 보증: AI의 환각 현상을 방지하기 위해 이메일 내 원문 링크 필수 포함 및 주의 문구 명시.

저작권: 뉴스 전문 복제가 아닌 '요약' 및 '링크' 제공 형태로 법적 이슈 최소화.
